# SK telecom - Kubernetes 기반 DataPlatform 개발 전문가

## 자기소개 (500자 이내)

배치와 스트리밍 파이프라인, 데이터 플랫폼 운영을 함께 해 온 8년차 데이터 엔지니어입니다. 가상자산 거래소에서는 Lakehouse 기반 플랫폼을 만들고, EKS/Helm/ArgoCD로 배포와 운영 기준을 맞췄습니다. Kafka Connect/Debezium/Flink/Iceberg로 준실시간 데이터 흐름도 구축했습니다. 제가 신경 쓰는 지점은 새 기능을 빨리 붙이는 일보다 오래 안정적으로 돌고, 문제를 관측할 수 있고, 동료가 다시 쓸 수 있는 플랫폼을 만드는 일입니다. 기술 선택은 ADR과 문서로 남기고, 장애와 품질 이슈는 팀이 함께 다룰 수 있는 기준으로 정리해 왔습니다.

## 직무경험 (1000자 이내)

Bithumb에서는 대용량 데이터를 수집/처리하는 Lakehouse 기반 데이터 플랫폼을 개발하고 운영했습니다. 기존 ASG/매니지드 서비스 중심 운영을 EKS로 옮기기 위해 클러스터를 구성했고, Helm Chart와 ArgoCD Application/ApplicationSet으로 배포 구조를 정리했습니다. ArgoCD, AWS Load Balancer Controller, Secrets Store CSI Driver, Datadog Agent Operator 등 플랫폼 운영 컴포넌트도 함께 정비해 환경별 설정과 Secret 참조 방식을 코드로 관리했습니다. 이후 Flink, Kafka Connect 같은 데이터 플랫폼 컴포넌트에도 같은 GitOps 운영 패턴을 적용할 수 있는 기반을 만들었습니다.

실시간 데이터 영역에서는 Kafka CDC 이벤트를 Flink Table API로 읽어 Iceberg 테이블에 적재하는 Sink 애플리케이션을 개발했습니다. changelog와 materialized 적재 모드를 나눠 원본 이벤트 보관과 최신 상태 테이블 구성을 함께 지원했고, Airflow DAG와 분리된 애플리케이션 빌드/배포 단위를 마련했습니다. Coinone에서는 Kinesis/Lambda 기반 실시간 적재 흐름을 MSK와 Kafka Connect로 전환하고, Debezium Source Connector를 활용한 CDC 파이프라인을 구축했습니다. connector 지연, 에러, offset 복구, binlog retention 같은 운영 지표를 Grafana와 알림으로 보며 중단 없는 데이터 수집 기준을 세웠습니다. DataHub, Airbyte, MageAI를 Kubernetes 환경에 배포해 메타데이터, 외부 수집, 워크플로우를 self-serve 형태로 제공한 경험도 있습니다.

## 지원동기 및 입사 후 포부 (1000자 이내)

SK텔레콤의 Kubernetes 기반 DataPlatform 개발 직무는 제가 최근 해 온 일과 바로 이어지는 역할이라고 봤습니다. 공고의 클라우드 네이티브 데이터 플랫폼, 대규모 Kubernetes 운영, 데이터 레이크 기술, 백엔드 API와 데이터 엔진 제공, 인증/인가와 거버넌스 설계는 단순한 파이프라인 개발보다 넓은 일입니다. 플랫폼 자체를 제품처럼 설계하고 운영해야 하는 영역에 가깝습니다. 저는 EKS 기반 데이터 플랫폼 전환, Kafka/Flink/Iceberg 기반 준실시간 파이프라인, DataHub와 Quality Gate를 통한 메타데이터/품질 관리 경험으로 이 문제를 실무에서 다뤄 왔습니다.

입사 후에는 먼저 기존 플랫폼의 컴포넌트, 배포 방식, 장애 대응 흐름, 데이터 흐름을 빠르게 파악하고 문서로 남기겠습니다. 그다음 Kubernetes 환경에서 데이터 엔진과 API를 안정적으로 제공할 수 있도록 Helm/GitOps 운영 기준, 관측성, 복구 절차, 설정 관리 방식을 다듬는 데 기여하고 싶습니다. 장기적으로는 Spark, Flink, StarRocks, Iceberg, Hive Metastore 같은 엔진과 레이크 기술을 사용자 요구에 맞게 조합하고, 인증/인가/품질/계보를 포함한 거버넌스 체계를 플랫폼 기능으로 녹여내고 싶습니다. 개발자, 인프라, 데이터 사용자 사이의 요구를 조율하며 SK텔레콤의 대규모 데이터를 더 믿고 쉽게 쓸 수 있는 플랫폼으로 만드는 엔지니어가 되겠습니다.

# SK telecom 경력기술 보조 문안

## Bithumb

### 담당업무

대용량 거래 데이터를 안정적으로 수집·처리하고 제공하는 Lakehouse 기반 데이터 플랫폼 개발과 운영을 맡았습니다. 기존 매니지드 서비스와 서버 중심으로 운영되던 데이터 플랫폼 컴포넌트를 EKS 기반 컨테이너 환경으로 옮기기 위해 Kubernetes 클러스터를 구성하고 Helm Chart 표준화, ArgoCD 기반 GitOps 배포 구조를 정리했습니다. Kafka Connect, Flink, Iceberg, Glue Catalog 등을 활용해 배치와 스트리밍 데이터 처리 경로를 설계했으며 CDC 이벤트를 Flink Table API로 처리해 Iceberg 테이블에 적재하는 준실시간 파이프라인을 개발했습니다. 내부 분석뿐 아니라 서비스와 마케팅 영역에서 활용할 수 있는 Serving Layer도 설계했습니다. SAD와 ADR로 주요 아키텍처 의사결정을 남겨 플랫폼의 유지보수성과 확장성을 높였습니다.

### 퇴직사유 제안

현재 재직 중이며, 퇴직보다는 이직 희망 사유로 작성하는 것이 적절합니다. 예시는 다음과 같습니다.

Kubernetes 기반 데이터 플랫폼 구축 경험을 더 큰 규모의 서비스와 데이터 환경에서 확장하고 싶어 지원했습니다. SK텔레콤의 대규모 데이터, 다양한 데이터 엔진, 플랫폼 API, 인증·인가와 거버넌스 체계는 제가 최근 집중해 온 클라우드 네이티브 데이터 플랫폼 개발 역량을 더 깊게 발휘할 수 있는 영역이라고 판단했습니다.

## Coinone

### 담당업무

가상자산 거래소에서 발생하는 서비스·거래 데이터를 클라우드 데이터웨어하우스에 안정적으로 적재했습니다. 임직원이 신뢰할 수 있는 데이터를 활용하도록 데이터 파이프라인과 분석 환경도 구축했습니다. Kinesis와 Lambda 기반의 기존 실시간 적재 구조를 MSK, Kafka Connect, Debezium 기반 CDC 파이프라인으로 전환했습니다. 데이터 수집 지연, 에러, offset 복구, binlog retention 같은 운영 지표는 Grafana와 알림으로 관리했습니다. 이벤트 메시지를 소비하는 스트리밍 파이프라인을 개발했습니다. 지표 저장소와 CLI 기반 지표 조회 도구를 만들어 합의된 비즈니스 지표를 재사용 가능한 데이터 프로덕트로 제공했습니다. DataHub, Airbyte, MageAI 등 메타데이터·외부 수집·워크플로우 도구를 Kubernetes 환경에 배포해 self-serve data 환경도 마련했습니다.

### 퇴직사유 제안

거래소 데이터 파이프라인과 self-serve data 환경 구축 경험을 쌓은 뒤, 더 큰 규모의 데이터 플랫폼을 설계·운영하는 역할로 성장하고자 이직했습니다. 단일 파이프라인 개발을 넘어 Lakehouse, Kubernetes, 실시간 처리, 데이터 품질과 거버넌스를 포함한 플랫폼 전반으로 책임 범위를 넓히고 싶었습니다.

## NHN Enterprise

### 담당업무

NHN Cloud의 검색·로그 분석 서비스인 Log & Crash Search 운영과 클라우드 서비스 배포 자동화 업무를 맡았습니다. Public Cloud 환경에서 서비스 안정성과 SLA를 유지하기 위해 운영 지표를 확인하고 장애 상황에 대응했으며 서비스에 사용되는 검색·로그 처리 오픈소스 컴포넌트의 특성과 운영 방식을 가까이에서 경험했습니다. NHN Cloud 서비스의 배포 파이프라인도 관리했습니다. CI/CD 흐름, 환경별 배포 설정, 릴리스 과정에 필요한 운영 절차를 유지보수하고 개발 조직의 요구사항을 반영해 배포 모듈을 개선했습니다. 서비스 변경이 운영 환경에 안정적으로 반영되도록 배포와 운영 사이의 연결 지점을 관리하며 클라우드 운영, 자동화, 장애 대응, 서비스 신뢰성에 대한 실무 감각을 쌓았습니다.

### 퇴직사유 제안

클라우드 서비스 운영과 배포 자동화 경험을 바탕으로 보다 데이터 중심의 플랫폼 개발 역할로 커리어를 확장하고자 이직했습니다. 운영 안정성과 자동화 역량을 데이터 파이프라인, 데이터웨어하우스, self-serve data 환경 구축에 적용하고 싶었습니다.

## TmaxData

### 담당업무

DBMS와 Big Data 제품의 기술 지원 및 고객사 PoC 수행을 맡았습니다. 고객 환경에서 발생하는 데이터베이스 성능, 설정, 장애, 제품 사용 이슈를 분석하고 원인을 파악해 해결 방안을 제시했습니다. 제품 도입 전 검증 과정에서는 요구사항을 확인하고 테스트 시나리오를 지원했습니다. 관계형 데이터베이스와 분산 데이터 처리 제품을 다루며 SQL, 데이터 모델, 저장 구조, 운영 설정의 기본기를 쌓았고, 고객과 개발 조직 사이에서 문제 상황을 정리해 전달하는 역할도 수행했습니다. 장애나 문의를 단순히 처리하는 데 그치지 않고 로그, 설정, 실행 결과를 바탕으로 원인을 좁혀 가며 데이터 기반 사고와 커뮤니케이션 역량을 키웠습니다.

### 퇴직사유 제안

DBMS와 Big Data 제품 기술지원 경험으로 데이터 시스템의 기본기를 쌓은 뒤, 고객 지원을 넘어 실제 서비스 운영과 클라우드 환경에서 시스템을 직접 관리하는 역할로 확장하고자 이직했습니다. 제품 단위의 이해를 서비스 운영, 자동화, 플랫폼 개발 경험으로 넓히고 싶었습니다.
