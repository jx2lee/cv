# 카카오 Data Analytics Engineer 자기소개

데이터는 많이 쌓이는 것보다, 신뢰할 수 있는 형태로 정리되어 실제 의사결정과 서비스 개선에 쓰일 때 더 큰 가치를 만든다고 생각합니다. 그 관점에서 커리어를 쌓아 왔습니다. 소프트웨어 벤더사와 클라우드 운영 조직, 그리고 가상자산 거래소에서 약 8년간 일하며 데이터 플랫폼 구축, 배치·스트리밍 파이프라인 운영, 데이터 품질 관리, 서비스 대상 데이터 서빙까지 폭넓게 경험했습니다. 커리어 초기에는 DBMS와 빅데이터 제품 기술 지원을 맡으며 문제를 구조적으로 진단하는 습관을 익혔고, 이후 클라우드 환경에서 서비스 운영과 배포 체계를 경험하며 안정성과 운영성의 중요성을 배웠습니다. 최근에는 Coinone과 Bithumb에서 데이터 엔지니어로 일하며 데이터웨어하우스, CDC, 이벤트 스트리밍, 셀프서브 데이터 환경, 데이터 API, 품질 게이트를 직접 설계하고 운영했습니다.

강점은 데이터를 단순히 적재하는 데서 멈추지 않고, 사용자가 실제로 활용할 수 있는 데이터셋과 흐름으로 바꾸는 데 있습니다. Coinone에서는 여러 조직이 공통으로 활용할 수 있는 지표 저장소와 셀프서브 데이터 환경을 운영하며, 데이터 조직에 요청하지 않아도 필요한 데이터를 스스로 찾고 활용할 수 있는 기반을 만들었습니다. Debezium과 Kafka Connect 기반 CDC 파이프라인을 구축할 때는 변경 이력을 안정적으로 확보하는 것에 그치지 않고, 지연·에러·복구를 관측할 수 있는 운영 체계를 함께 설계해 월 기준 99.9% 적재 성공률을 유지했습니다. Bithumb에서는 데이터 품질 게이트를 도입해 파이프라인 실패율을 30% 낮췄고, DW 집계 결과를 서비스에서 활용할 수 있도록 Data API와 Serving Layer를 설계하며 데이터 결과물이 실제 사용자 경험으로 이어지는 과정까지 다뤘습니다.

SQL과 Python을 중심으로 문제를 해결해 왔고, Airflow 기반 배치 파이프라인을 설계·운영하며 기존 DAG 수정부터 신규 작성까지 직접 수행해 왔습니다. 또한 Redshift, BigQuery, Kafka, EKS, Helm, ArgoCD 등 다양한 데이터·플랫폼 환경을 다뤄 왔기 때문에, 데이터 모델링과 파이프라인 구현뿐 아니라 운영 과정에서 발생하는 제약까지 함께 고려하는 편입니다. 특히 중요하게 보는 것은 데이터 품질, 관측 가능성, 복구 가능성입니다. 데이터는 한 번 만들어 놓는 것으로 끝나지 않고, 다양한 이해관계자가 지속적으로 믿고 사용할 수 있어야 한다고 생각합니다.

카카오 제품분석엔지니어링팀에 끌리는 이유도 여기에 있습니다. 분석 조직과 제품 조직이 실제로 필요로 하는 데이터셋을 설계하고, 그 데이터가 안정적으로 활용될 수 있도록 플랫폼과 도구까지 함께 고민하는 역할을 지향합니다. 제품을 이해한 데이터 모델링, 사용 목적이 분명한 데이터 마트 설계, 품질과 안정성을 갖춘 파이프라인 운영, 그리고 여러 이해관계자와 공통의 언어로 협업하는 방식은 지금까지 해 온 일과도 자연스럽게 이어집니다. 카카오에서는 데이터를 더 잘 쌓는 사람보다, 데이터를 더 잘 쓰이게 만드는 엔지니어로 기여하고 싶습니다.
